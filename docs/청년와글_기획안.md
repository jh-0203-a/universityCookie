# 청년와글 (Youth-Wagle) 서비스 기획안

> 20·30 청년들의 동네 생활 플랫폼
> _작성일 2026.06.14 · 현재까지 구현된 프로토타입 기준_

---

## 1. 한 줄 소개

**대학·지역 인증 배지를 기반으로 한, 20~30대 청년 생활 플랫폼.**
커뮤니티 · 모임 · 스터디 · 중고거래 · 채팅을 한 앱에서.

---

## 2. 기획 배경 & 방향 전환

처음에는 "졸업생 동문 전용(대학생쿠키)" 커뮤니티로 출발했지만,
타깃을 **"20~30대 청년 전체"** 로 확장했습니다.

| 구분 | 이전 (대학생쿠키) | 현재 (청년와글) |
|------|------------------|----------------|
| 타깃 | 졸업생 동문만 | 20·30 청년 누구나 |
| 인증 | **필수 관문** (인증해야 기능 사용) | **선택 + 배지** (안 해도 다 됨) |
| 인증 종류 | 졸업증명서 1종 | 대학🎓 / 지역📍 2종 |
| 정체성 | 학교 동문 커뮤니티 | 동네 청년 생활 플랫폼 |

**핵심 결정**: 인증은 더 이상 기능을 막지 않습니다.
누구나 글쓰기·모임·중고거래·채팅을 쓸 수 있고, 인증한 사람은 **닉네임 옆에 배지**가 붙어 신뢰도를 보여줍니다.

---

## 3. 타깃 사용자

- **주 타깃**: 수도권(서울·경기·인천)에 사는 20~30대 청년
- 대학생 / 졸업생 / 직장인 / 취준생 등 신분 무관
- 동네에서 사람을 만나고(모임), 함께 공부하고(스터디), 물건을 거래하고(중고), 정보를 나누고 싶은(커뮤니티) 사람

---

## 4. 핵심 컨셉 — 인증 배지 시스템

인증은 **선택사항**이며, 완료 시 닉네임 옆에 작은 배지로 표시됩니다.

### 4-1. 대학 인증 🎓
대학 재학생/졸업생임을 증명. (서류는 관리자가 심사)

- **메인 수단** (가장 확실): `재학증명서`, `졸업증명서`
  - 증명서 선택 시 **DocuLink 진위확인** 사이트 바로가기 제공
- **보조 수단** (메인 서류가 없을 때): `동문회 이메일`, `졸업앨범 사진`, `학생증(졸업생 표시)`

### 4-2. 지역 인증 📍
해당 지역 거주자임을 증명.

- **수단**: `주민등록증`
- ⚠️ **개인정보 보호**: 주민등록증 제출 시 **주민번호 뒷자리(생년월일 뒤 7자리)는 반드시 가리고** 촬영. 가리지 않은 서류는 거절.

### 4-3. 심사 흐름
```
회원: 인증 종류 선택(대학/지역) → 수단 선택 → 사진 첨부 → 제출
        ↓
관리자: 인증 요청 목록에서 서류 확인 → 승인 / 거절
        ↓
승인 시: 대학 → 🎓 배지 / 지역 → 📍 배지  (닉네임 옆 자동 표시)
```

## 5. 주요 기능

| 기능 | 설명 |
|------|------|
| **홈** | 우리 지역 글 + 인기 글 모아보기 |
| **커뮤니티** | 게시판(자유·익명·학교별·취업 + 사용자 생성), 글/댓글, 좋아요(1인 1회), 익명 작성, 사진 첨부, 신고 |
| **모임** | 모임 개설/참여, 모임 게시판(공지), 단체 채팅, 가입 인사 |
| **스터디** | 모임의 한 형태로 운영 (예: 코딩 스터디) |
| **중고거래** | 상품 등록(판매중/예약중/판매완료), 사진, 판매자 표시 |
| **채팅 / 쪽지** | 1:1 실시간 채팅 + 비실시간 쪽지함 |
| **알림** | 댓글·좋아요·쪽지 등 알림 |
| **프로필** | 내 정보, 인증 관리, 프로필 사진 |

## 6. 지역(권역) 구조

자유 입력 대신 **시/도 → 권역 → 학교** 단계 선택으로 데이터를 깔끔하게 유지.
**현재는 수도권만 오픈**, 지방은 차츰 확대 예정 (앱에 안내 문구 노출).

| 시/도 | 권역 |
|------|------|
| **서울** | 서울 동부 / 서울 서부 |
| **경기** | 북부권 / 남동부권 / 남서부권 |
| **인천** | 인천권 (통합) |

> 권역 안에서는 일반대·여대·신학대·전문대·대학원대 구분 없이 학교를 하나로 통합.
> (데이터: `src/regions.ts`)

## 8. 운영 · 관리자 기능

관리자(운영자) 계정은 다음을 처리:

- **인증 심사**: 제출된 대학/지역 인증 서류 확인 → 승인/거절
- **회원 관리**: 정지(1달~5년)/탈퇴 처리
- **신고 처리**: 게시글·댓글·회원·상품·모임 신고 확인
- **문의 답변**: 1:1 문의 응대

---

## 9. 데이터 모델 (핵심)

- **UserAccount**: 닉네임, 이메일, 학교, 지역, `verified`(대학🎓), `regionVerified`(지역📍), 상태(정상/정지/탈퇴), 역할(회원/관리자)
- **VerificationRequest**: `type`(대학/지역), `method`(수단), 서류 사진, 상태(대기/승인/거절)
- **Post / Comment / Meetup / MarketItem / ChatRoom / Note / Report / Inquiry / Notification**

> 닉네임 → 인증 여부 조회: `badgesOf(nickname)` → `{school, region}`
> 배지 표시 컴포넌트: `<VerifiedBadges nickname={...} />`

---

## 10. 기술 스택

- **프론트엔드**: React 19 + TypeScript + Vite + Tailwind CSS
- **상태/저장**: 프로토타입은 `localStorage` (목업), 백엔드는 **PostgreSQL**
- **서버**: Express (`server/index.js`) — 회원가입 등 API
- **사진**: 폴더에 파일 저장 + DB에는 경로만 저장
- **실행**: `npm run dev` (http://localhost:3000)
